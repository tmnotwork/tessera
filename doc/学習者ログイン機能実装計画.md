# 学習者ログイン機能 実装計画

## 概要

現在のTesseraアプリは認証なし（匿名アクセス）で動作している。
学習者がユーザーIDでログインできるようにするための実装計画をまとめる。

---

## 現状の課題

| 項目 | 現状 |
|------|------|
| 認証 | なし（Supabase匿名アクセス） |
| ユーザーテーブル | なし |
| ロール管理 | UI上の切り替えのみ（DB上の区別なし） |
| RLSポリシー | 匿名ユーザーに全CRUD許可（開発用設定） |

## 実装タスク一覧

### TASK-1: DBマイグレーション

**ファイル**: `supabase/migrations/00012_add_learners.sql`

#### 1-1. `learner_profiles` テーブルの作成

```sql
CREATE TABLE learner_profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  user_id TEXT UNIQUE NOT NULL,   -- 学習者が使うログインID（例: student01）
  display_name TEXT,               -- 表示名
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

- `id` はSupabase Authの `auth.users.id` に対応
- `user_id` は教師が発行する短い識別子（例: `student01`）
- ログイン時はメールアドレスの代わりに `user_id` を使う（後述）

#### 1-2. RLSポリシーの更新

**コンテンツ系テーブル（subjects, knowledge, memorization_cards, questions 等）**

```sql
-- 既存の anon 全許可ポリシーを削除し、認証済みユーザーのみ読み取り可に変更
DROP POLICY IF EXISTS "anon_all" ON subjects;
CREATE POLICY "authenticated_read" ON subjects
  FOR SELECT TO authenticated USING (true);
```

**learner_profiles テーブル**

```sql
-- 学習者は自分自身のプロフィールのみ参照・更新可
ALTER TABLE learner_profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "self_read" ON learner_profiles
  FOR SELECT TO authenticated USING (auth.uid() = id);

CREATE POLICY "self_update" ON learner_profiles
  FOR UPDATE TO authenticated USING (auth.uid() = id);
```

**教師（管理者）ロールの扱い**

- Supabase の `service_role` キーを教師側アプリで使用する
- または `app_metadata` に `role: teacher` を付与し、RLSで判定する

### TASK-2: Supabase Auth の設定

**作業場所**: Supabase管理コンソール（ブラウザ）

#### 2-1. Email認証の有効化

- Authentication > Providers > Email を有効化
- 「Confirm email」は運用に応じてOFF推奨（教師管理の場合）

#### 2-2. 学習者アカウントの登録方式

学習者は短い `user_id`（例: `student01`）でログインするが、Supabase Authはメール形式を要求するため、以下の変換ルールを採用する。

```
user_id: "student01"
→ email: "student01@tessera.local"
→ password: 教師が設定した任意のパスワード
```

- この変換はアプリ内で自動的に行い、学習者はIDとパスワードのみ入力する

#### 2-3. テストアカウントの作成

Supabase管理コンソール > Authentication > Users で手動作成、または後述の教師用管理画面から作成する。

### TASK-3: Flutter アプリ - 認証状態管理

**ファイル**: `lib/src/app_scope.dart`

#### 3-1. `AppScope` に認証情報を追加

```dart
// 追加するフィールド
User? get currentUser => Supabase.instance.client.auth.currentUser;
bool get isLoggedIn => currentUser != null;

// ログイン
Future<void> loginLearner(String userId, String password) async {
  final email = '$userId@tessera.local';
  await Supabase.instance.client.auth.signInWithPassword(
    email: email,
    password: password,
  );
}

// ログアウト
Future<void> logoutLearner() async {
  await Supabase.instance.client.auth.signOut();
}
```

#### 3-2. 認証状態の監視

```dart
// アプリ起動時にセッションを復元
Supabase.instance.client.auth.onAuthStateChange.listen((data) {
  // ログイン/ログアウトに応じてUIを更新
  notifyListeners();
});
```

### TASK-4: Flutter アプリ - ログイン画面の作成

**新規ファイル**: `lib/src/screens/learner_login_screen.dart`

#### 画面構成


#### 実装内容

- エラー時（ID/パスワード不一致）はスナックバーでメッセージ表示
- ローディング中はボタンをスピナーに切り替える

### TASK-5: Flutter アプリ - ナビゲーション制御の更新

**ファイル**: `lib/main.dart`

#### 5-1. 学習タブの表示制御

### TASK-6: 教師側 - 学習者管理UI

**ファイル**: `lib/main.dart`（`TeacherAdminPage` に追加）

#### 6-1. 学習者一覧タブの追加

教師管理画面に「学習者管理」タブを追加する。

| 機能 | 内容 |
|------|------|
| 学習者一覧 | `learner_profiles` を一覧表示 |
| 学習者追加 | ユーザーID・表示名・パスワードを入力してSupabase Authにユーザー作成 |
| パスワードリセット | 任意のパスワードに変更 |
| 学習者削除 | Supabase Authからユーザーを削除 |

#### 6-2. 学習者追加の実装

教師側は `service_role` キーまたはAdmin APIを使ってユーザーを管理する。

> **注意**: 学習者の作成・削除にはSupabase Admin APIが必要なため、教師用アプリは別途 `SUPABASE_SERVICE_ROLE_KEY` を使用する、またはSupabase管理コンソールで手動管理する。
