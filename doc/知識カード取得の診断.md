# 知識カードが表示されないときの診断

## 1. Supabase にカードがあるか

- 英文法（subject_id: `72cba8cc-28b9-41fb-bf72-257a99139831`）に紐づく `knowledge` 行は Supabase に存在する。
- MCP または Dashboard の SQL で次を実行して確認できる:
  ```sql
  SELECT id, subject_id, unit, left(content, 40) FROM public.knowledge WHERE subject_id = '72cba8cc-28b9-41fb-bf72-257a99139831' LIMIT 5;
  ```

## 2. テストで anon 取得を確認

```bash
flutter test test/knowledge_fetch_test.dart
```

- **成功**: anon で `knowledge` を subject_id 指定で取得できている。API とデータは問題なし。
- **失敗**: レスポンス本文や statusCode から API/RLS の原因を切り分ける。

## 3. アプリ実行時のデバッグプリント

`flutter run` で起動し、**ログイン後**に「学習」→「知識を学ぶ」→「英文法」を開く。

コンソールに次のいずれかが出る（Debug ビルド時のみ）:

| ログ | 意味 |
|------|------|
| `[KnowledgeListScreen._load] subjectId=..., dataSource=Supabase` | 取得元は Supabase。subjectId が正しいか確認。 |
| `[KnowledgeListScreen._load] subjectId=..., dataSource=LocalDB` | 取得元はローカル DB。同期で取り込まれているか確認。 |
| `[KnowledgeListScreen._load] Supabase try1: select with knowledge_card_tags join` | join ありの 1 回目を実行。 |
| `[KnowledgeListScreen._load] Supabase try1 FAILED: ...` | join が失敗（例: knowledge_card_tags / knowledge_tags の RLS や存在）。続けて try2 が走る。 |
| `[KnowledgeListScreen._load] Supabase try2 ok: count=N` | join なしで取得成功。N が 0 なら RLS で 0 件になっている可能性。 |
| `[KnowledgeListScreen._load] FAILED: ...` | 両方失敗。例外メッセージとスタックで原因を特定。 |
| `[KnowledgeRepositorySupabase.getBySubject] try1 failed ...` | リポジトリ側で try1 が失敗。同上。 |
| `[KnowledgeRepositorySupabase.getBySubject] subjectId=... result count=N` | リポジトリの結果件数。 |

## 4. 原因の切り分け

- **try1 だけ失敗し try2 で count > 0**: join 先（knowledge_card_tags / knowledge_tags）の RLS またはテーブル未作成。表示はされるがタグが付かない可能性。
- **try2 でも count = 0**: 認証済みでの `knowledge` の SELECT が RLS で 0 件になっている。`knowledge: learner read` が authenticated に付いているか確認。
- **FAILED で例外**: メッセージが PGRST205 ならテーブルなし、403/401 なら認証・RLS を確認。
