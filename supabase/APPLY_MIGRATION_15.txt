================================================================
マイグレーション 00015 適用手順（Supabase CLI）
対象: question_knowledge に is_core フラグを追加
================================================================

【前提】
- Supabase CLI がインストール済みであること
- プロジェクトルート（learning_platform/）で作業すること

【このプロジェクトのプロジェクト ref】
  wnufzrehvhcwclnwxwim
（.env の SUPABASE_URL のホスト名部分。ダッシュボードの
 「Project Settings > General > Reference ID」でも確認可能）

----------------------------------------------------------------
ステップ 1: ログイン（未ログインの場合のみ）
----------------------------------------------------------------

  supabase login

ブラウザが開くので、Supabase アカウントでログイン。
※ リンクや push の前に必ず完了しておくこと。

----------------------------------------------------------------
ステップ 2: 初期化（config.toml がない場合のみ）
----------------------------------------------------------------

  supabase init

supabase/ に config.toml が無いときだけ実行する。
既に migrations/ などがある場合は init で上書きされない。

----------------------------------------------------------------
ステップ 3: プロジェクトとリンク（未リンクの場合のみ）
----------------------------------------------------------------

  supabase link --project-ref wnufzrehvhcwclnwxwim

別のプロジェクトを使う場合は <YOUR_PROJECT_REF> を
ダッシュボードの「Reference ID」に置き換える。

----------------------------------------------------------------
ステップ 4: 現在の適用済みマイグレーション確認（任意）
----------------------------------------------------------------

  supabase migration list

リモートに 00014 まで適用済みで、00015 が pending になっていれば OK。

----------------------------------------------------------------
ステップ 5: マイグレーション適用
----------------------------------------------------------------

  supabase db push

00015_add_is_core_to_question_knowledge.sql が適用される。

【適用される内容】
  ALTER TABLE public.question_knowledge
    ADD COLUMN IF NOT EXISTS is_core BOOLEAN NOT NULL DEFAULT false;

  既存レコードは全て is_core = false（非コア）として扱われる。
  問題作成時に明示的に true をセットすることでコア問題に指定する。

----------------------------------------------------------------
ステップ 6: 適用確認
----------------------------------------------------------------

  supabase migration list

00015 が applied になっていれば完了。

または Supabase ダッシュボード > Table Editor > question_knowledge で
is_core カラムが追加されていることを確認。

----------------------------------------------------------------
クイック実行（ログイン・リンク済みの場合）
----------------------------------------------------------------

  1. supabase migration list    ※ 状態確認（任意）
  2. supabase db push           ※ 00015 を適用
  3. supabase migration list    ※ applied 確認

----------------------------------------------------------------
注意事項
----------------------------------------------------------------

- ADD COLUMN IF NOT EXISTS を使っているため、
  誤って2回実行してもエラーにはならない。

- 既存の question_knowledge レコードの is_core は全て false になる。
  コア問題への変更は、アプリの問題編集画面から行う
  （UI実装は別途対応）。
