================================================================
マイグレーション 00015 適用手順（Supabase CLI）
対象: question_knowledge に is_core フラグを追加
================================================================

【前提】
- Supabase CLI がインストール済みであること
- tessera/ ディレクトリで作業すること

----------------------------------------------------------------
ステップ 1: ログイン（未ログインの場合のみ）
----------------------------------------------------------------

  supabase login

ブラウザが開くので、Supabase アカウントでログイン。

----------------------------------------------------------------
ステップ 2: プロジェクトとリンク（未リンクの場合のみ）
----------------------------------------------------------------

  supabase link --project-ref <YOUR_PROJECT_REF>

<YOUR_PROJECT_REF> は Supabase ダッシュボードの
「Project Settings > General > Reference ID」で確認できる。
（例: abcdefghijklmnop）

config.toml がない場合は先に初期化が必要:

  supabase init
  supabase link --project-ref <YOUR_PROJECT_REF>

----------------------------------------------------------------
ステップ 3: 現在の適用済みマイグレーション確認（任意）
----------------------------------------------------------------

  supabase migration list

リモートに 00014 まで適用済みで、00015 が pending になっていれば OK。

----------------------------------------------------------------
ステップ 4: マイグレーション適用
----------------------------------------------------------------

  supabase db push

00015_add_is_core_to_question_knowledge.sql が適用される。

【適用される内容】
  ALTER TABLE public.question_knowledge
    ADD COLUMN IF NOT EXISTS is_core BOOLEAN NOT NULL DEFAULT false;

  既存レコードは全て is_core = false（非コア）として扱われる。
  問題作成時に明示的に true をセットすることでコア問題に指定する。

----------------------------------------------------------------
ステップ 5: 適用確認
----------------------------------------------------------------

  supabase migration list

00015 が applied になっていれば完了。

または Supabase ダッシュボード > Table Editor > question_knowledge で
is_core カラムが追加されていることを確認。

----------------------------------------------------------------
注意事項
----------------------------------------------------------------

- ADD COLUMN IF NOT EXISTS を使っているため、
  誤って2回実行してもエラーにはならない。

- 既存の question_knowledge レコードの is_core は全て false になる。
  コア問題への変更は、アプリの問題編集画面から行う
  （UI実装は別途対応）。
