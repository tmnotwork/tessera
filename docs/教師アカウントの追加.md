# 教師アカウントを増やす手順

教師は Supabase **Auth のメールユーザー**であり、`profiles.role = teacher` になります。学習者（`@tessera.local`）とは別経路です。

## 1. アプリから新規登録（いちばん手軽）

1. アプリの **教材管理** タブ → **新規登録**。
2. 許可されたメールアドレスとパスワードを入力。
3. **確認メール**のリンクを踏んでから、**ログイン**タブでログイン。

### 本番で「開発者ドメインだけ」にしたい場合

ビルド時に許可ドメインを渡します（カンマ区切り）。**未設定・空のときは制限なし**（ローカル開発向け）。

```bash
flutter build apk --dart-define=TEACHER_SIGNUP_ALLOWED_DOMAINS=yourcompany.com,example.org
```

- サブドメインも許可されます（例: `mail.yourcompany.com` は `yourcompany.com` が許可されていれば可）。
- **ログイン**は従来どおり、既存の教師アカウントならドメイン外メールでも可能です。ブロックされるのは **新規登録**だけです。

> クライアント側のチェックなので、厳密な禁止は Supabase の「Auth 設定（メール確認・レート制限）」や将来のサーバー側フックと組み合わせると安全です。

## 2. Supabase Dashboard から作る

1. **Authentication → Users → Add user**。
2. メール・パスワード（または magic link 運用）を設定。
3. `profiles` は `auth.users` 作成トリガーで `teacher` が付く想定です。行が無い場合は `ensure_my_profile` 相当の運用か、手動で `profiles` を確認してください。

アプリの「新規登録」を閉じたい場合でも、**ダッシュボードならいつでもユーザーを追加**できます。

## 3. 運用の決め方（目安）

| やりたいこと | おすすめ |
|-------------|----------|
| チームにメールを配って各自登録 | **1** + `TEACHER_SIGNUP_ALLOWED_DOMAINS` |
| アプリから教師登録を完全に隠したい | **2** のみ（アプリの新規登録タブは非表示にする変更は別途） |
| 単体テスト・ローカル | define なし（制限なし） |

新しい教師を増やすたびに **やること**は、(1) の場合は「その人にアプリで新規登録してもらう」、(2) の場合は「Dashboard でユーザーを1件追加」です。どちらも **別の ID 体系は不要**で、既存の `profiles.role` と RLS がそのまま使えます。
